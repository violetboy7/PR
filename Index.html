Goals & Fit

게임 테스트 엔지니어로서 지향하는 역할과 성장 방향

“시스템을 이해하는 QA가 더 좋은 테스트를 설계합니다.”

단순히 체크리스트를 수행하는 QA가 아니라, 기획·개발·유저 경험 사이의 연결 구조를 이해하고 결함이 발생하기 전 리스크를 먼저 정의하는 QA가 되고자 합니다. 데이터 기반 검증, 자동화, 라이브 서비스 이해도를 통해 팀이 신뢰할 수 있는 품질 기준을 만드는 것이 목표입니다.

Risk First고위험 결함과 유저 손실 가능성을 먼저 식별하는 우선순위 중심 검증
System Thinking기능 단위가 아니라 경제·보상·플랫폼까지 연결된 시스템 관점 분석
Tool Maker반복 검증을 자동화하고 팀 전체가 재사용할 수 있는 표준 도구 제작

학습 및 성장 방향

실무 역량 확장을 위해 지속적으로 탐구·학습하고 있는 영역

CI/CD

CI/CD 개발 파이프라인

빌드 단계부터 결함을 조기 차단하기 위해 CI/CD 파이프라인에 QA가 어떻게 결합되는지 학습하고 있습니다. 자동 빌드·배포·테스트 흐름을 이해하여 라이브 서비스의 안정성을 확보하는 것이 목표입니다.

AGILE

애자일 테스팅

스프린트 주기 내에서 점진적으로 품질을 확보하는 애자일 테스팅 방식을 학습 중입니다. 변화하는 요구사항에 대응하면서도 회귀 테스트 범위를 효과적으로 관리하는 전략을 익히고자 합니다.

AUTO

테스트 자동화 (API · 웹 · 클라이언트)

AutoHotkey 기반 반복 자동화 경험을 토대로, API·웹·클라이언트 자동화 도구로 영역을 확장하고자 합니다. 검증 범위를 넓히고 회귀 비용을 줄이는 자동화 체계 구축이 다음 목표입니다.

LLM

LLM 활용한 게임 테스팅 패러다임 변화

LLM이 테스트 케이스 생성, 버그 리포트 분석, 시나리오 탐색 등 QA 업무 흐름을 어떻게 바꾸는지 주목하고 있습니다. 사람과 AI가 협업하는 테스팅 모델을 실험적으로 학습하며 변화에 대비하고 있습니다.
